Neural Network Architecture Study
Overview

Systematic comparison of multi-layer perceptron (MLP) configurations for supervised classification on a tabular sensor dataset. Variables under study: network depth (2–4 hidden layers), layer width (32–256 neurons) and activation function (ReLU, Sigmoid, Tanh). Goal: identify the architecture with best accuracy/training-cost tradeoff.

94.2%
Best test accuracy
12
Configs compared
3×
ReLU convergence speed
Methodology
1
Dataset preparation — tabular sensor readings (voltage, current, temperature) split 70/15/15 train/val/test. Standard-score normalisation applied per feature.
2
Architecture grid search — depth ∈ {2,3,4}, width ∈ {32,64,128,256}, activation ∈ {ReLU, Sigmoid, Tanh}. All trained for 100 epochs with Adam (lr=1e-3).
3
Regularisation study — Dropout (p=0.3) and L2 weight decay (λ=1e-4) applied to the best-performing topology; val loss variance reduced by 28 %.
4
Loss curve analysis — training and validation loss plotted per epoch; early stopping triggered at patience=10 to prevent overfitting.
🔬 In progress

Best configuration so far: 3-hidden-layer ReLU MLP (128–64–32), test accuracy 94.2 %. Next: experiment with batch normalisation and learning rate scheduling (CosineAnnealingLR).

Electronics · Arduino · Measurement
RC Filter Characterisation
Overview

Experimental study of RC and LC passive filter circuits — from breadboard prototyping to microcontroller-based automated frequency sweeps. Bode plots generated from ADC measurements; cutoff frequencies compared to theoretical values.

1.4%
τ measurement error
6
RC configs tested
10 kSps
ADC sample rate
Methodology
1
Breadboard validation — low-pass RC (R = 1 kΩ, C = 100 nF, f_c = 1.59 kHz) driven by function generator. Rise time measured on oscilloscope; matched theory within 2 %.
2
Automated sweep (Arduino) — sketch drives DAC output through 10 Hz–20 kHz in 50-point log steps; ADC records Vout amplitude. Python plots Bode magnitude.
3
High-pass & band-pass — same board reconfigured; cutoff frequencies confirmed at f_c = 1/2πRC. Band-pass centre frequency tuned by LC tank resonance.
4
PCB migration — filter network moved to dedicated 2-layer PCB to eliminate breadboard parasitic capacitance (~5 pF/node) corrupting HF measurements.
✅ Status — completed

All 6 RC configurations characterised. Time constant τ extracted from transient fits: max deviation 1.4 % from R·C. Full Bode plots exported as PDF. PCB v1 manufactured and verified.

KiCad · Hardware · PCB Design
Test Board Design v1
Overview

First full PCB design cycle: schematic capture → layout → Gerber export → JLCPCB manufacture → bench test. The board provides a clean, repeatable platform for passive filter experiments, replacing error-prone breadboard setups.

2
Layer count
48
Components placed
0
DRC errors at fab
Design Details
1
Schematic (KiCad) — voltage divider chain (10 kΩ ladder), decoupling network (100 nF X7R per rail), SMA edge connectors for signal input/output.
2
Layout — ground plane on layer 2 for low-impedance return path. Signal traces kept < 50 mm to minimise parasitic inductance at test frequencies.
3
Design rule check — JLCPCB 2-layer rules applied: min trace 0.127 mm, min clearance 0.127 mm, via drill 0.3 mm. Zero DRC violations before Gerber export.
4
Bench test — continuity check with DMM, then powered at 3.3 V / 5 V. All voltage rails within ±1 % of target. No shorts detected.
✅ Status — completed

Board manufactured (JLCPCB, 5-day lead time), assembled and validated. Parasitic capacitance on signal nodes reduced from ~5 pF (breadboard) to < 0.5 pF (PCB), enabling reliable measurements above 50 kHz.

ESP32 · Embedded C · FreeRTOS
ESP32 Sensor Data-Logger
Overview

FreeRTOS-based firmware for the ESP32 that continuously samples a multi-channel ADC, writes timestamped records to SD card (FAT32) and streams a live plot to a Python serial dashboard. Designed as a reusable data-acquisition backbone for future hardware experiments.

10 kSps
Sample rate
4
ADC channels
12-bit
ADC resolution
Firmware Architecture
1
Sampling task — high-priority FreeRTOS task reads ADC1 channels at 10 kSps using timer ISR. Samples pushed to a lock-free ring buffer.
2
SD-write task — low-priority task drains the ring buffer in 512-byte blocks to SD card (SPI, SDMMC). CSV format: timestamp_us, ch0, ch1, ch2, ch3.
3
Serial stream — every 100 ms, a summary packet (min/max/mean per channel) is sent over UART at 921 600 baud for live Python dashboard plotting.
4
Python dashboard — Matplotlib animation reads serial port; scrolling 5-second window of all 4 channels rendered in real time.
🔬 In progress

Core sampling and SD-write tasks verified; no buffer overruns at 10 kSps with 4 channels. Next: add Wi-Fi OTA update support and a web dashboard to replace the serial cable.
