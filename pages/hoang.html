Scintillator Housing Optimisation
Overview

Design and optimisation of a parametric housing for LYSO scintillator crystals (3×3×5 mm³) coupled to SiPM detectors. The goal is to maximise photon collection efficiency while keeping the geometry machinable on a standard CNC mill.

+18%
Photon yield (est.)
v2
Current revision
4
Geometry variants tested
Methodology
1
Parametric CAD (OpenSCAD) — housing dimensions driven by variables; wall thickness, reflector gap and crystal clearance all configurable without touching geometry code.
2
Cylinder → cube migration — previous cylindrical design left dead corners with no reflective coverage. Cube form closes this gap and simplifies milling.
3
Geant4 optical simulation — 10⁶ optical photons generated isotropically inside the crystal. Photon detection probability scored at the SiPM active surface for each geometry variant.
4
Surface finish study — ESR mirror film vs white Teflon tape vs bare Al compared. ESR + cube combination yields highest yield in simulation.
📐 Key result

Cube geometry with ESR reflector film: estimated photon collection probability rises from 31.4 % (cylinder baseline) to 37.0 % — a relative improvement of 17.8 %. Prototype fabrication scheduled Q3 2025.

C++ · ROOT · SAMPIC
Waveform Analysis Pipeline
Overview

A fully automated ROOT/C++ pipeline that processes raw binary output from the SAMPIC waveform digitiser: binary frame decoding → baseline subtraction → charge integration → multi-channel histogram export. Reduces per-run analysis time from ~3 h (manual) to under 8 min.

~22×
Speed-up vs manual
64
Channels supported
0.4 mV
Noise floor σ
Pipeline Steps
1
Binary parsing — SAMPIC frame header decoded; sample arrays extracted into ROOT TTree per channel per event.
2
Baseline correction — first 10 samples averaged per waveform; baseline vector subtracted sample-by-sample.
3
Charge integration — trapezoidal integration over a configurable gate window [t₀, t₀+Δt]. Gate optimised per channel.
4
Histogram rendering — h_sampic[64] array filled; multi-layer overlay canvas drawn with custom TLegend and colour palette.
✅ Status — completed

Pipeline deployed and validated on real testbeam data. Charge spectra for 16 SiPM channels rendered in a single ROOT macro call. Output: PDF canvas + ROOT file with all histograms stored.

Geant4 · C++ · MC Simulation
Optical Photon MC Simulation
Overview

Full Geant4 simulation of optical photon transport inside a scintillator–reflector–SiPM assembly. Used to predict photon detection efficiency, timing jitter and the impact of surface finish before committing to physical prototypes.

10⁶
Photons / run
4
Surface models tested
~14 min
Run time (8 cores)
Simulation Details
1
Geometry construction — G4Box / G4OpticalSurface definitions for crystal, air gap, reflector and SiPM sensitive volume.
2
Physics list — G4OpticalPhysics with Rayleigh scattering, absorption, and boundary processes. Refractive indices from NIST material database.
3
Scoring — sensitive detector on SiPM face records hit time and wavelength. Output written to ROOT TTree for offline analysis.
🔬 In progress

Currently tuning optical surface roughness parameters (σ_alpha) to match bench-top measurements. Next step: timing resolution study as a function of crystal length (5 mm → 20 mm).
